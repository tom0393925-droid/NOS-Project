# プロジェクトの基本方針

- 指示を出す人間（Tomoya）はプロジェクトマネージャーであり、コードの文法ではなく仕様を管理します。

# Claude Codeへの厳格なルール

1. **ファイルの読み込み制限**: 指示されたタスクに関連する最小限のファイルのみを読み込むこと。プロジェクト全体の無駄なスキャンはトークン消費を防ぐため禁止する。

   ファイル読み込みルール
   - 機能追加・小修正 → 最小限だけ読む
   - 削除・リファクタ → 対象ファイルは全体を読む
   - バグ修正 → エラー箇所前後100行程度

2. **細かい実装の徹底（One PR = One Feature）**: 複数の要件を一度に実装せず、1つの機能や修正が終わるごとに動作確認のステップを挟むこと。

3. **作業後の人間語レポート**: コードの修正が完了したら、作業終了前に必ず以下のフォーマットでレポートを出力すること。

   - 【変更したファイル】:
   - 【何を変えたか】: プログラミング用語を避け、非エンジニアにわかる言葉で3行以内で要約。
   - 【確認手順】: 画面上で修正が成功したかを確認するための具体的な操作手順。

   作業後レポートと同時にREADME も更新する。

4. **ブランチ運用**: ブランチを作らずmainブランチで直接作業する。

5. **自動プッシュ**: コードを変更したらgit add → commit → push origin main まで確認なしで自動実行する。

6. **プレビュー不要**: ファイル編集後にプレビューパネルへの言及をしない。
